→
index.html → server.js GET + X-PAYMENT
⑤ 서명 포함 재요청

server.js 내부 처리 VERIFY
⑥ 서명 검증 + 잔액 확인
// ① ecrecover (오프체인) ethers.verifyMessage(message, signature) → "0xAD2B...D533" ✓ // ② eth_call (온체인 읽기) provider.getBalance("0xAD2B...D533") → 1.55 ETH ≥ 0.0015 ETH ✓

// 설명
핵심 개념 (Key Concepts)
// 01
HTTP 서명 vs 트랜잭션
③④ 단계의 서명은 HTTP 메시지 서명입니다. 블록체인에 기록되지 않고 가스비도 없습니다. 실제 트랜잭션은 ⑦번 한 번뿐입니다.
OFF-CHAIN
// 02
ecrecover
서명값과 원본 메시지만으로 서명자 주소를 수학적으로 복원합니다. 개인키 없이도 "누가 서명했는지" 검증 가능한 것이 핵심입니다.
CRYPTOGRAPHY
// 03
402 상태코드
1999년 HTTP/1.1 스펙에 예약만 되고 26년간 사용되지 않았습니다. Coinbase가 2025년에 이를 블록체인 결제 표준으로 부활시켰습니다.
